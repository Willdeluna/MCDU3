# Pilot Review Rubric

Use this rubric after Phase 0 baseline capture and at each phase gate. Scores are 1-5, where 1 is not useful/realistic and 5 is close enough for serious procedure practice.

## Session Metadata

| Field                 | Value                                   |
| --------------------- | --------------------------------------- |
| Date                  |                                         |
| Reviewer role         | Pilot / instructor / simmer / developer |
| Aircraft familiarity  | Boeing 737 NG / Airbus A320 / both      |
| Device                | Desktop / iPad / simulator rig          |
| App version or commit |                                         |
| Scenario flown        |                                         |

## Scored Categories

| Category                       | Score | Notes                                                                                              |
| ------------------------------ | ----: | -------------------------------------------------------------------------------------------------- |
| Boeing visual realism          |       | Font, display grid, colors, inverse video, key layout                                              |
| Boeing page flow realism       |       | IDENT, POS INIT, RTE, DEP/ARR, PERF, TAKEOFF REF                                                   |
| Scratchpad behavior            |       | Error wording, clearing, invalid entries, EXEC staging                                             |
| LEGS/HOLD/FIX realism          |       | Waypoint edits, holds, fix rings, constraints                                                      |
| ND training clarity            |       | Does the ND make route edits, fixes, holds, discontinuities, and approach settings understandable? |
| CLB/CRZ/DES/PROG usefulness    |       | Trainer-level predictions and cues                                                                 |
| Airbus visual realism          |       | MCDU colors, labels, page grouping                                                                 |
| Airbus scope honesty           |       | Display-only vs functional pages are clear                                                         |
| MSFS connected-mode usefulness |       | Latency, stability, diagnostics, PMDG round trip                                                   |
| iPad cockpit usability         |       | Touch size, layout stability, brightness, orientation                                              |
| Tutorial/training usefulness   |       | Guidance, hints, scoring, confidence gain                                                          |
| Overall training confidence    |       | Would recommend for procedure practice                                                             |

## Required Gate Scores

| Gate                        | Required result                         |
| --------------------------- | --------------------------------------- |
| Phase 1 Boeing visuals      | Average visual realism >= 4.0/5, n >= 4 |
| Phase 2 Boeing procedures   | Procedure realism >= 4.2/5              |
| Phase 3 Airbus polish       | Airbus visual rating >= 4.0/5           |
| Phase 5 MSFS connected mode | Sim training value >= 4.3/5             |
| Phase 6 curriculum          | Training effectiveness >= 4.3/5         |
| End-state                   | Pilot usefulness >= 4.5/5               |

## Review Notes

- Reviewers should identify the three most realism-breaking issues before suggesting nice-to-have polish.
- Capture screenshots or video clips for visual concerns when possible.
- Do not mark a phase gate passed if the reviewer only inspected screenshots and did not operate the CDU/MCDU.
