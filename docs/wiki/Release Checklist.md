# Release Checklist

Before release:

- Run the validation commands in `docs/RELEASE_CHECKLIST.md`.
- Update `docs/STATUS.md` with command evidence.
- Update `CHANGELOG.md`.
- Review `docs/KNOWN_LIMITATIONS.md`.
- Run relevant visual baselines when UI changes.
- Run `npm run measure:visual`.
- Record PMDG/MSFS live evidence only if tested in the required Windows/MSFS/PMDG environment.
- Verify public docs do not overstate certification, hardware accuracy, or live validation.
